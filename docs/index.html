What is Meta‑C?

An OOP language with clean syntax that compiles directly to C. Designed for games, real-time systems, and any domain where performance and memory control matter.

Compiles to C

Your .mc code becomes readable C — then gcc turns it into a native binary. No VM, no interpreter, no overhead.

Explicit Memory

No garbage collector, no stack for user data. Everything lives in named blocks you control. Reset a block — all allocations go with it.

Hot Reload

Swap code without stopping your program. Powered by dlopen + inotify. Edit, save, see results instantly.

Zero Magic

What you write is what gets generated. Transparent compilation with #line directives so you debug in Meta‑C, not C.

Code Example

Structs, blocks, methods — all the pieces working together.

Meta‑C (.mc) 
package BLOCKS_DEMO

block global = 256MB
block game   = 64MB
block temp   = 8MB

struct Player {
    int hp
    int ammo
    String name

    fn Player(int h, int a, String n) {
        hp = h
        ammo = a
        name = n
    }

    fn take_damage(int dmg) {
        hp -= dmg
        if hp < 0 { hp = 0 }
    }
}

struct Enemy {
    int hp
    int damage

    fn Enemy(int h, int d) {
        hp = h
        damage = d
    }

    fn attack(Player target) {
        target.take_damage(damage)
    }
}

fn main() {
    Player p = Player(100, 30, "Felipe") @game
    Enemy e = Enemy(50, 10) @game
    e.attack(p)

    while p.hp > 0 {
        e.attack(p)
    }

    temp.reset()
    game.reset()
    global.reset()
}

Features

Everything you need for high-performance systems programming — nothing you don't.

⚡

OOP with Structs

Classes? No — structs with methods, constructors, and interface support. Clean GDScript-inspired syntax that compiles to C structs and functions.

🧱

Block Memory

Declare named memory blocks (8KB – 256MB+). All allocations go into blocks. No malloc/free per object — just block.reset() to wipe everything.

🔄

Hot Reload

Edit code while it runs. dlopen + inotify detect changes, reload .so files, remap functions. Zero downtime iteration for games and live systems.

📈

Bump Allocator

Block memory uses a bump allocator — a pointer bump and done. No fragmentation, no free list, no syscall per allocation. Predictable microsecond latency.

🐛

GDB Integration

Step through your original .mc code, not the generated C. #line directives map every line back. Pretty-printers show Meta‑C types natively.

📊

TUI Visualizer

Ncurses-based dashboard showing block memory usage, allocations, and hot reload status in real time. Built for developers who live in the terminal.

🔌

VS Code Extension

Syntax highlighting, breakpoints in .mc files, and a memory-view panel that plots block allocations. Debug your memory layout visually.

🌍

Cross-platform

Write once, compile anywhere C goes — Linux, macOS, Windows. The runtime is pure C with POSIX dlopen. No platform lock-in, no runtime dependency.

Quick Start

From zero to running in four steps.

1

Clone

Get the repository and enter the project directory.

git clone https://github.com/anomalyco/meta-c.git
cd meta-c
2

Build

Meta‑C uses SCons. Build the compiler and runtime in one command.

scons
3

Write

Create a .mc file — for example the player demo above — with your favorite editor.

touch game.mc
4

Compile & Run

The compiler translates .mc → C, then gcc produces the binary. The runtime links automatically.

./meta-c game.mc -o game
./game
